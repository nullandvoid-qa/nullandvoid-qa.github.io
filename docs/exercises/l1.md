# Exercício: O que é QA e por que importa (l1)

Objetivo: praticar identificação de edge cases e formular respostas operacionais.

Exercício:

1. Liste 3 edge cases que um usuário poderia enfrentar ao usar um app de login.
2. Para cada edge case, descreva o comportamento esperado e como você validaria (passos de teste).

Gabarito exemplo:

- Edge case 1: Login sem internet
  - Resultado esperado: mensagem de erro clara e opção de tentar novamente.
  - Validação: desligar rede e tentar login; verificar mensagem e estado do botão de envio.

- Edge case 2: Senha com caracteres especiais
  - Resultado esperado: aceitar caracteres UTF-8 e validar corretamente.
  - Validação: usar senhas com símbolos e verificar autenticação.

- Edge case 3: Clique repetido no botão enviar
  - Resultado esperado: botão desabilitado durante requisição para evitar duplicação.
  - Validação: simular cliques rápidos e checar que apenas uma requisição é enviada.
