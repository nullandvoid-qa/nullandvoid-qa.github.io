# Roadmap Prioritário de Melhorias (Resumo)

Objetivo: elevar as aulas amostradas ao padrão competitivo, reduzir gaps nas trilhas e preparar o site para tradução completa.

Prioridade Alta (0-2 semanas)
- A1 — Exercícios com gabarito para cada aula amostrada (10 aulas)
  - Descrição: adicionar 1 exercício prático por aula + gabarito e instruções de correção.
  - Estimativa: 2h por aula (20h total)
  - Responsável sugerido: conteúdo / mentor técnico

- A2 — Incluir material de apoio e repositório de exemplos
  - Descrição: adicionar links, arquivos de exemplo e repositórios para cada aula crítica.
  - Estimativa: 1h por aula (10h)
  - Dependência: A1

- A3 — Marcar metadados de revisão (autor, data, status)
  - Descrição: padronizar cabeçalho de metadata em cada lição (YAML/JSON inline).
  - Estimativa: 0.5h por aula (5h)

Prioridade Média (2-4 semanas)
- B1 — Templates editáveis (checklists, SDLC diagram, matriz de risco)
  - Estimativa: 8h

- B2 — Exemplos de scripts/infra para automação e regressão
  - Estimativa: 8-12h

Prioridade Baixa (4+ semanas)
- C1 — Produção de material multimídia (vídeos curtos) para aulas prioritárias
  - Estimativa: produção por vídeo 4-8h (roteiro + gravação básica)

Milestones
- M1 (Fim semana 1): concluir A1 (5 aulas) + A2 (5 aulas) + A3 baseline
- M2 (Fim semana 2): concluir A1 e A2 restantes + publicar revisão de progresso
- M3 (Fim semana 4): completar B1 e iniciar B2

Entregáveis imediatos criados por este plano
- Checklist de qualidade (`docs/QUALITY_CHECKLIST.md`) — já presente
- Relatório de lacunas (`reports/tracks-gap-plan.md`) — já gerado
- Relatório de amostras e revisão (`reports/lesson-samples.md`, `reports/lesson-review.md`) — já gerados

Próximas ações recomendadas (curto prazo)
- Iniciar coleta de feedback com 5 usuários beta (recrutar e criar formulário de feedback).
- Alocar 2 sprints curtos para A1/A2 com responsáveis claros.
