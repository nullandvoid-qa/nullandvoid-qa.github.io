<h2>Android Emulator para Testes</h2>

<h3>Objetivos de Aprendizado</h3>
<p>Após esta aula, você será capaz de:</p>
<ul>
  <li>Configurar e usar o Android Emulator (AVD)</li>
  <li>Entender as diferenças entre emulator e device real</li>
  <li>Executar testes automatizados no Android Emulator</li>
  <li>Diagnosticar problemas comuns no emulador Android</li>
</ul>

<h3>Resumo Executivo</h3>
<p>O Android Emulator (também conhecido como AVD - Android Virtual Device) é a ferramenta oficial do Android para testar apps sem dispositivos físicos. Diferente do iOS Simulator, ele roda em múltiplas plataformas (Windows, macOS, Linux) e oferece configurações altamente customizáveis. É essencial para desenvolvimento e testes de apps Android, cobrindo a maioria dos cenários com excelente suporte a automação.</p>

<h3>Configuração Inicial</h3>

<h4>Requisitos</h4>
<ul style="margin:1rem 0">
  <li><strong>Android Studio</strong> (disponível para Windows, macOS, Linux)</li>
  <li><strong>Java Development Kit (JDK)</strong> - 11 ou superior</li>
  <li><strong>HAXM ou Hyper-V</strong> - para aceleração de hardware (Windows/macOS)</li>
  <li><strong>8GB RAM mínimo</strong> (16GB recomendado)</li>
</ul>

<h4>Instalação</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# 1. Baixe e instale o Android Studio
# https://developer.android.com/studio

# 2. Durante a instalação, selecione:
# - Android SDK
# - Android SDK Platform-Tools
# - Android SDK Build-Tools
# - Android Emulator

# 3. Configure variáveis de ambiente (Linux/macOS)
export ANDROID_HOME=$HOME/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/tools/bin
export PATH=$PATH:$ANDROID_HOME/platform-tools

# 4. Verifique a instalação
adb version
emulator -version
</pre>

<h3>Criando e Gerenciando AVDs</h3>

<h4>Android Studio AVD Manager</h4>
<p>O AVD Manager é a interface gráfica para criar e gerenciar emuladores:</p>
<ol style="margin:1rem 0; list-style-position:inside">
  <li>Abra Android Studio → Tools → Device Manager</li>
  <li>Clique em "Create Device"</li>
  <li>Escolha um device (Pixel 6, Pixel 7, etc)</li>
  <li>Selecione a system image (Android 13, 14, etc)</li>
  <li>Configure as configurações do AVD</li>
  <li>Clique em "Finish"</li>
</ol>

<h4>Linha de Comando (avdmanager & emulator)</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Listar system images disponíveis
sdkmanager --list | grep system-images

# Baixar uma system image específica
sdkmanager "system-images;android-34;google_apis;x86_64"

# Criar um AVD via linha de comando
avdmanager create avd -n "pixel_6_api_34" \
  -k "system-images;android-34;google_apis;x86_64" \
  -d "pixel_6"

# Listar AVDs criados
emulator -list-avds

# Iniciar um AVD específico
emulator -avd pixel_6_api_34

# Iniciar com opções avançadas
emulator -avd pixel_6_api_34 \
  -no-snapshot-load \
  -wipe-data \
  -no-boot-anim \
  -gpu auto \
  -memory 4096 \
  -cores 4
</pre>

<h3>Testes Automatizados no Android Emulator</h3>

<h4>Com Espresso (Framework Oficial)</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Adicionar dependência no build.gradle
androidTestImplementation 'androidx.test.espresso:espresso-core:3.5.1'
androidTestImplementation 'androidx.test:runner:1.5.2'
androidTestImplementation 'androidx.test:rules:1.5.0'

# Rodar testes no emulator
./gradlew connectedAndroidTest

# Rodar testes em um AVD específico
./gradlew connectedAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=com.example.MyTest

# Rodar teste específico
./gradlew connectedAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=com.example.MyTest#testMySpecificTest
</pre>

<h4>Com Appium (Cross-Platform)</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# capabilities para Android Emulator
{
  "platformName": "Android",
  "platformVersion": "14",
  "deviceName": "pixel_6_api_34",
  "automationName": "UiAutomator2",
  "app": "/path/to/app.apk",
  "noReset": true,
  "avd": "pixel_6_api_34",
  "avdLaunchTimeout": 180000
}

# Comando para iniciar Appium
appium --address 127.0.0.1 --port 4723

# Exemplo de teste em Python
from appium import webdriver

caps = {
    "platformName": "Android",
    "platformVersion": "14",
    "deviceName": "pixel_6_api_34",
    "automationName": "UiAutomator2",
    "app": "/path/to/app.apk",
    "avd": "pixel_6_api_34"
}

driver = webdriver.Remote("http://127.0.0.1:4723/wd/hub", caps)
</pre>

<h4>Com ADB Direto</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Verificar devices conectados
adb devices

# Instalar um app
adb install /path/to/app.apk

# Reinstalar mantendo dados
adb install -r /path/to/app.apk

# Desinstalar app
adb uninstall com.package.name

# Launchar um app
adb shell am start -n com.package.name/.MainActivity

# Verificar logs
adb logcat

# Filtrar logs por tag
adb logcat -s TAG_NAME

# Tirar screenshot
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png
</pre>

<h3>Configurações Avançadas</h3>

<h4>Simular Condições de Rede</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Usando o emulator console
telnet localhost 5554

# Comandos disponíveis no console:
network speed edge      # Simular 2G
network speed gsm       # Simular 2.5G
network speed full      # Velocidade completa
network delay 2000     # Adicionar 2000ms de latência

# Via adb
adb shell svc wifi disable      # Desabilitar WiFi
adb shell svc data disable      # Desabilitar dados móveis
adb shell svc wifi enable       # Habilitar WiFi
adb shell svc data enable       # Habilitar dados móveis
</pre>

<h4>Simular Localização</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Definir localização específica
adb shell am start -a android.intent.action.VIEW \
  -d "geo:37.7749,-122.4194"

# Usando adb para setar location
adb shell settings put secure location_providers_allowed gps,network

# Via Extended Controls no Android Studio
# Emulator → Extended Controls → Location
</pre>

<h4>Simular Bateria e Status</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">

<h3>Troubleshooting Comum</h3>

<div style="background:#fef3c7; padding:1.5rem; border-radius:0.5rem; margin:1.5rem 0; border-left:4px solid #f59e0b">
<p><strong>Problema 1: Emulator não inicia</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se a virtualização está habilitada no BIOS</li>
  <li>No Windows: habilite Hyper-V ou instale HAXM</li>
  <li>No macOS: verifique se HAXM está instalado corretamente</li>
  <li>Aumente a RAM alocada para o emulator</li>
</ul>

<p><strong>Problema 2: Emulator muito lento</strong></p>
<ul style="margin:1rem 0">
  <li>Use system images x86_64 em vez de ARM</li>
  <li>Habilite GPU acceleration: -gpu auto</li>
  <li>Aumente RAM e CPU cores no AVD config</li>
  <li>Use snapshots para boot rápido</li>
</ul>

<p><strong>Problema 3: ADB não encontra o emulator</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se o adb server está rodando: <code>adb start-server</code></li>
  <li>Reinicie o adb: <code>adb kill-server && adb start-server</code></li>
  <li>Verifique a porta: emulator usa 5554+ por padrão</li>
</ul>

<p><strong>Problema 4: App crasha no emulator mas não no device</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se a arquitetura do app é compatível (x86 vs ARM)</li>
  <li>Alguns apps não funcionam em emuladores por segurança</li>
  <li>Use adb logcat para ver o crash log</li>
</ul>
</div>

<h3>Melhores Práticas</h3>
<ul style="margin:1rem 0">
  <li><strong>Use snapshots</strong> - para boot rápido do emulator</li>
  <li><strong>Teste em múltiplas Android versions</strong> - Android 10, 11, 12, 13, 14</li>
  <li><strong>Use x86_64 system images</strong> - são mais rápidas que ARM</li>
  <li><strong>Habilite GPU acceleration</strong> - para performance gráfica</li>
  <li><strong>Automate setup</strong> - scripts para criar e configurar AVDs</li>
  <li><strong>Use headless mode em CI/CD</strong> - <code>-no-window</code></li>
</ul>
