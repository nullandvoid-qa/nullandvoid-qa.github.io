<h2>Testes funcionais vs não-funcionais</h2>

<h3>🎯 Objetivos de Aprendizado</h3>
<p>Após esta aula, você será capaz de:</p>
<ul>
  <li>Diferenciar testes funcionais de não-funcionais</li>
  <li>Listar exemplos práticos de cada tipo</li>
  <li>Identificar como escolher o tipo correto para um cenário</li>
  <li>Planejar um conjunto equilibrado de testes para uma feature</li>
</ul>

<h3>📊 Resumo Executivo</h3>
<p>Testes funcionais validam o <strong>comportamento</strong> do sistema: se ele faz o que promete. Testes não-funcionais validam <strong>como</strong ele faz: desempenho, segurança, usabilidade, acessibilidade e escalabilidade.</p>

<h3>🔍 Testes Funcionais</h3>
<ul style="margin:1rem 0">
  <li><strong>O que são:</strong> Testam a funcionalidade do produto.</li>
  <li><strong>Pergunta-chave:</strong> "Isso funciona como esperado?"</li>
  <li><strong>Exemplos:</strong> login, cadastro, compra, busca, filtros.</li>
  <li><strong>Técnicas:</strong> casos de uso, critérios de aceitação, equivalência.</li>
</ul>

<h3>⚙️ Testes Não-Funcionais</h3>
<ul style="margin:1rem 0">
  <li><strong>Performance:</strong> tempo de resposta, carga, stress.</li>
  <li><strong>Segurança:</strong> autenticação, autorização, proteção de dados.</li>
  <li><strong>Usabilidade:</strong> facilidade de uso, clareza, navegação.</li>
  <li><strong>Acessibilidade:</strong> compatibilidade com leitores de tela, contraste, navegação por teclado.</li>
  <li><strong>Confiabilidade:</strong> estabilidade sob condições reais.</li>
</ul>

<h3>📌 Exemplo na Prática</h3>
<p>Feature: checkout de um e-commerce</p>
<ul>
  <li><strong>Funcional:</strong> usuário consegue adicionar item ao carrinho, editar quantidade e finalizar compra.</li>
  <li><strong>Performance:</strong> a página de checkout carrega em menos de 2 segundos.</li>
  <li><strong>Segurança:</strong> dados do cartão são enviados via HTTPS e nunca aparecem no front-end.</li>
  <li><strong>Usabilidade:</strong> o formulário de endereço é claro e não confunde o usuário.</li>
  <li><strong>Acessibilidade:</strong> labels são associadas aos campos e o teclado percorre todos os controles.</li>
</ul>

<h3>✏️ Exercício Prático</h3>
<p>Para uma feature simples, escreva:</p>
<ol style="margin:1rem 0; list-style-position:inside">
  <li>2 testes funcionais</li>
  <li>1 teste de performance</li>
  <li>1 teste de segurança</li>
  <li>1 teste de usabilidade ou acessibilidade</li>
</ol>

<h3>✅ Checklist Prático de Testes por Tipo</h3>
<div style="background:#f0fdf4; padding:1.5rem; border-radius:0.5rem; margin:1.5rem 0; border-left:4px solid #10b981">
<p><strong>Funcional:</strong></p>
<ul style="margin:1rem 0">
  <li>✓ Todos os requisitos básicos funcionam?</li>
  <li>✓ Casos de borda foram testados?</li>
  <li>✓ Integrações com outros sistemas funcionam?</li>
</ul>

<p><strong>Performance:</strong></p>
<ul style="margin:1rem 0">
  <li>✓ Tempo de resposta está dentro do SLA?</li>
  <li>✓ Sistema suporta pico de usuários?</li>
  <li>✓ Performance é consistente em diferentes dispositivos?</li>
</ul>

<p><strong>Segurança:</strong></p>
<ul style="margin:1rem 0">
  <li>✓ Dados sensíveis são criptografados?</li>
  <li>✓ Autenticação e autorização funcionam corretamente?</li>
  <li>✓ Sistema é vulnerável a ataques comuns (SQL injection, XSS)?</li>
  <li>✓ Headers de segurança estão configurados (CSP, HSTS, X-Frame-Options)?</li>
  <li>✓ Senhas são armazenadas com hash + salt adequados?</li>
</ul>

<p><strong>Usabilidade:</strong></p>
<ul style="margin:1rem 0">
  <li>✓ Interface é intuitiva para novos usuários?</li>
  <li>✓ Mensagens de erro são claras e úteis?</li>
  <li>✓ Fluxo principal é fácil de completar?</li>
</ul>

<p><strong>Acessibilidade:</strong></p>
<ul style="margin:1rem 0">
  <li>✓ Funciona com leitores de tela (NVDA, JAWS, VoiceOver)?</li>
  <li>✓ Pode ser navegado apenas por teclado (Tab, Enter, Esc)?</li>
  <li>✓ Contraste de cores é suficiente (WCAG AA 4.5:1)?</li>
  <li>✓ Imagens têm atributos alt descritivos?</li>
  <li>✓ Formulários têm labels associadas corretamente?</li>
  <li>✓ Focus indicators são visíveis em todos os elementos interativos?</li>
</ul>
</div>

<h3>🛠️ Ferramentas Recomendadas por Tipo de Teste</h3>
<div style="background:#dbeafe; padding:1.5rem; border-radius:0.5rem; margin:1.5rem 0; border-left:4px solid #3b82f6">
<p><strong>Para Acessibilidade:</strong></p>
<ul style="margin:1rem 0">
  <li><strong>axe DevTools</strong> - Extensão de browser que detecta violações WCAG automaticamente</li>
  <li><strong>WAVE</strong> - Ferramenta visual de avaliação de acessibilidade em páginas web</li>
  <li><strong>Lighthouse</strong> - Auditoria automática inclui módulo de acessibilidade</li>
  <li><strong>NVDA (Windows) / VoiceOver (Mac)</strong> - Leitores de tela para testes manuais</li>
  <li><strong>Color Contrast Analyzer</strong> - Verifica contraste de cores em tempo real</li>
</ul>

<p><strong>Para Segurança:</strong></p>
<ul style="margin:1rem 0">
  <li><strong>OWASP ZAP</strong> - Scanner de segurança gratuito e open-source</li>
  <li><strong>Burp Suite</strong> - Ferramenta profissional de testes de segurança web</li>
  <li><strong>SSL Labs</strong> - Testa configuração SSL/TLS do servidor</li>
  <li><strong>Security Headers</strong> - Verifica headers de segurança HTTP</li>
  <li><strong>Nmap</strong> - Scanner de rede para identificar portas e serviços vulneráveis</li>
</ul>

<p><strong>Para Performance:</strong></p>
<ul style="margin:1rem 0">
  <li><strong>Lighthouse</strong> - Auditoria de performance web (Core Web Vitals)</li>
  <li><strong>WebPageTest</strong> - Teste de performance detalhado com múltiplas localizações</li>
  <li><strong>k6</strong> - Ferramenta de teste de carga moderna baseada em scripts</li>
  <li><strong>JMeter</strong> - Ferramenta clássica de teste de carga e performance</li>
  <li><strong>Chrome DevTools Performance</strong> - Análise de runtime e profiling</li>
</ul>
</div>

<h3>🎯 Exemplo Prático: Teste de Acessibilidade</h3>
<p>Para testar se um formulário de login é acessível:</p>
<ol style="margin:1rem 0">
  <li><strong>Com leitor de tela:</strong> Navegue pelo formulário usando apenas teclado. Verifique se o leitor anuncia "campo de email, obrigatório" ao focar no input.</li>
  <li><strong>Com axe DevTools:</strong> Execute a auditoria. Corrija violações como "inputs sem labels" ou "baixo contraste".</li>
  <li><strong>Com Color Contrast Analyzer:</strong> Verifique se o texto "Esqueci minha senha" tem contraste mínimo de 4.5:1 com o fundo.</li>
  <li><strong>Teste de teclado:</strong> Tente completar o fluxo usando Tab, Enter e Esc. Verifique se não há teclas presas.</li>
</ol>

<h3>🔒 Exemplo Prático: Teste de Segurança Básico</h3>
<p>Para testar segurança de um formulário de login:</p>
<ol style="margin:1rem 0">
  <li><strong>Teste de SQL Injection:</strong> Tente inserir `' OR '1'='1` no campo de usuário. Se o login funcionar, há vulnerabilidade grave.</li>
  <li><strong>Teste de XSS:</strong> Tente inserir `<script>alert('XSS')</script>` em campos de texto. Se o script executar, há vulnerabilidade.</li>
  <li><strong>Verificação de HTTPS:</strong> Certifique-se que todos os formulários usam HTTPS e não aceitam HTTP.</li>
  <li><strong>Headers de segurança:</strong> Verifique se o servidor envia headers como `Content-Security-Policy`, `X-Frame-Options`, `Strict-Transport-Security`.</li>
  <li><strong>Lockout de conta:</strong> Tente fazer login com senha errada 5 vezes. A conta deve ser temporariamente bloqueada.</li>
</ol>
