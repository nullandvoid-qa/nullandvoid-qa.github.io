<h3>Resumo Executivo</h3>
<p>BrowserStack é uma plataforma de testes baseada em nuvem que permite testar websites e aplicações mobile em milhares de browsers e dispositivos reais. Diferente de emuladores locais, o BrowserStack oferece acesso a dispositivos físicos reais (iPhones, Androids, iPads) e browsers reais (Safari, Chrome, Firefox, Edge) sem precisar comprar ou manter nenhum hardware. É ideal para testes cross-browser e cross-device, especialmente para equipes sem orçamento para comprar muitos dispositivos.</p>

<h3>Configuração Inicial</h3>

<h4>Requisitos</h4>
<ul style="margin:1rem 0">
  <li><strong>Conta BrowserStack</strong> (plano gratuito disponível para testes)</li>
  <li><strong>Access Key</strong> (fornecida no dashboard do BrowserStack)</li>
  <li><strong>Framework de teste</strong> (Selenium, Cypress, Playwright, Appium, etc)</li>
  <li><strong>Conexão internet estável</strong> (testes rodam na nuvem)</li>
</ul>

<h4>Obtendo Credenciais</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">

# 2. Navegue para Account → Access Key
# 3. Copie seu username e access key

# Exemplo de credenciais:
BROWSERSTACK_USERNAME = "your_username"

<h3>Testes Web no BrowserStack</h3>

<h4>Configuração Selenium</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Selenium
pip install selenium

# Exemplo em Python
from selenium import webdriver
from selenium.webdriver.common.desired_capabilities import DesiredCapabilities

# Configuração básica
caps = {
    "browserName": "chrome",
    "browserVersion": "latest",
    "os": "Windows",
    "osVersion": "11",
    "buildName": "Selenium 4",
    "sessionName": "My first test"
}

# Inicializar driver
driver = webdriver.Remote(
    command_executor=f"https://{BROWSERSTACK_USERNAME}:{BROWSERSTACK_ACCESS_KEY}@hub-cloud.browserstack.com/wd/hub",
    desired_capabilities=caps
)

# Executar teste
driver.get("https://www.example.com")
print(driver.title)

# Fechar driver
driver.quit()
</pre>

<h4>Configuração Cypress</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Cypress BrowserStack plugin
npm install --save-dev cypress-browserstack-plugin

# cypress.config.js
const { defineConfig } = require("cypress");
const browserstack = require("cypress-browserstack-plugin");

module.exports = defineConfig({
  e2e: {
    setupNodeEvents(on, config) {
      config = browserstack(config);
      return config;
    },
    specPattern: "cypress/e2e/**/*.cy.{js,jsx,ts,tsx}",
    env: {
      browserstack_username: process.env.BROWSERSTACK_USERNAME,
      browserstack_access_key: process.env.BROWSERSTACK_ACCESS_KEY,
    },
  },
});

# browserstack.json
{
  "username": "your_username",
  "access_key": "your_access_key",
  "browsers": [
    {
      "browser": "chrome",
      "os": "Windows 11",
      "versions": ["latest"]
    },
    {
      "browser": "safari",
      "os": "OS X",
      "versions": ["latest"]
    }
  ],
  "run_settings": {
    "cypress_config_file": "./cypress.config.js",
    "cypress_version": "latest",
    "project_name": "My Cypress Project",
    "build_name": "Build #1"
  }
}

<h4>Configuração Playwright</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Playwright
npm install @playwright/test

# playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  projects: [
    {
      name: 'browserstack-chrome',
      use: {
        connectOptions: {
          wsEndpoint: `wss://cdp.browserstack.com/playwright?caps=${encodeURIComponent(JSON.stringify({
            'browser': 'chrome',
            'browser_version': 'latest',
            'os': 'Windows',
            'os_version': '11',
            'browserstack.username': process.env.BROWSERSTACK_USERNAME,
            'browserstack.accessKey': process.env.BROWSERSTACK_ACCESS_KEY,
          }))}`,
        },

    },
  ],
});

<h3>Testes Mobile no BrowserStack</h3>

<h4>Configuração Appium para Android</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Appium
pip install Appium-Python-Client

# Exemplo em Python
from appium import webdriver

# Configuração para Android real device
caps = {
    "platformName": "android",
    "deviceName": "Google Pixel 7",
    "platformVersion": "13.0",
    "app": "bs://sample.app",  # ou upload local
    "build": "Android App Test",
    "name": "First Android Test",
    "browserstack.username": BROWSERSTACK_USERNAME,
    "browserstack.accessKey": BROWSERSTACK_ACCESS_KEY,
    "browserstack.idleTimeout": 300,
    "browserstack.debug": True
}

# Inicializar driver
driver = webdriver.Remote(
    "http://hub-cloud.browserstack.com/wd/hub",
    caps
)

# Executar teste
element = driver.find_element("id", "com.example.app:id/button")
element.click()

# Fechar driver
driver.quit()
</pre>

<h3>Funcionalidades Avançadas</h3>

<h4>Parallel Testing</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Rodar testes em paralelo em múltiplos browsers
browsers = [
    {"browser": "chrome", "os": "Windows", "os_version": "11"},
    {"browser": "firefox", "os": "Windows", "os_version": "11"},
    {"browser": "safari", "os": "OS X", "os_version": "Ventura"},
    {"browser": "edge", "os": "Windows", "os_version": "10"}
]

# Usar threading ou multiprocessing para paralelismo
from concurrent.futures import ThreadPoolExecutor

def run_test(browser_config):
    caps = {**browser_config, **common_caps}
    driver = webdriver.Remote(hub_url, caps)
    # executar teste
    driver.quit()

with ThreadPoolExecutor(max_workers=4) as executor:
    executor.map(run_test, browsers)
</pre>

<h3>Troubleshooting Comum</h3>

<div style="background:#fef3c7; padding:1.5rem; border-radius:0.5rem; margin:1.5rem 0; border-left:4px solid #f59e0b">
<p><strong>Problema 1: Session falha com "Invalid credentials"</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se username e access key estão corretos</li>
  <li>Regenere a access key no dashboard se necessário</li>
  <li>Verifique se não há espaços extras nas credenciais</li>
</ul>

<p><strong>Problema 2: Testes muito lentos</strong></p>
<ul style="margin:1rem 0">
  <li>Use parallel testing para reduzir tempo total</li>
  <li>Reduza número de screenshots/videos se não crítico</li>
  <li>Use browsers mais rápidos (Chrome vs Safari)</li>
  <li>Escolha regions mais próximas geograficamente</li>
</ul>

<p><strong>Problema 3: App não instala no device</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se o app é compatível com o OS version</li>
  <li>Certifique-se de que o app está assinado corretamente</li>
  <li>Use o upload da API em vez de path local</li>
  <li>Verifique se o device está disponível no plano</li>
</ul>

<p><strong>Problema 4: Local testing não funciona</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se o BrowserStack Local está rodando</li>
  <li>Certifique-se de que a capability local está habilitada</li>
  <li>Verifique firewalls e configurações de rede</li>
  <li>Teste connectivity: ping browserstack.com</li>
</ul>
</div>

<h3>Melhores Práticas</h3>
<ul style="margin:1rem 0">
  <li><strong>Use parallel testing</strong> - reduza tempo de execução drasticamente</li>
  <li><strong>Escolha devices estratégicos</strong> - não teste todos, teste os mais usados</li>
  <li><strong>Use builds e tags</strong> - organize testes por feature/sprint</li>
  <li><strong>Habilite debug apenas quando necessário</strong> - aumenta custo</li>
  <li><strong>Combine com local testing</strong> - para ambientes de desenvolvimento</li>
  <li><strong>Monitore quotas</strong> - evite exceder limites do plano</li>
</ul>
