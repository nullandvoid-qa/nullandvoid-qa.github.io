<h2>SauceLabs para Testes</h2>

<h3>Objetivos de Aprendizado</h3>
<p>Após esta aula, você será capaz de:</p>
<ul>
  <li>Entender o que é SauceLabs e seus benefícios</li>
  <li>Configurar e usar SauceLabs para testes web</li>
  <li>Configurar e usar SauceLabs para testes mobile</li>
  <li>Integrar SauceLabs em pipelines de CI/CD</li>
  <li>Comparar SauceLabs com BrowserStack</li>
</ul>

<h3>Resumo Executivo</h3>
<p>SauceLabs é uma plataforma de testes baseada em nuvem similar ao BrowserStack, permitindo testar websites e aplicações mobile em milhares de browsers e dispositivos reais. Fundado em 2008, foi um dos pioneiros em testes em nuvem e oferece forte integração com ferramentas de CI/CD. Seu foco principal é em automação de testes e DevOps, com excelente suporte para testes contínuos e integração com ferramentas como Jenkins, GitHub Actions, e CircleCI.</p>

<h3>Configuração Inicial</h3>

<h4>Requisitos</h4>
<ul style="margin:1rem 0">
  <li><strong>Conta SauceLabs</strong> (plano gratuito disponível para testes)</li>
  <li><strong>Username e Access Key</strong> (fornecidos no dashboard do SauceLabs)</li>
  <li><strong>Framework de teste</strong> (Selenium, Cypress, Playwright, Appium, etc)</li>
  <li><strong>Conexão internet estável</strong> (testes rodam na nuvem)</li>
</ul>

<h4>Obtendo Credenciais</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# 1. Crie uma conta em https://saucelabs.com
# 2. Navegue para User Settings → Access Key
# 3. Copie seu username e access key

# Exemplo de credenciais:
SAUCE_USERNAME = "your_username"
SAUCE_ACCESS_KEY = "your_access_key_abcdef123456"

# Ou usando variáveis de ambiente
export SAUCE_USERNAME="your_username"
export SAUCE_ACCESS_KEY="your_access_key"
</pre>

<h3>Testes Web no SauceLabs</h3>

<h4>Configuração Selenium</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Selenium
pip install selenium

# Exemplo em Python
from selenium import webdriver
from selenium.webdriver.common.desired_capabilities import DesiredCapabilities

# Configuração básica
caps = {
    "platformName": "Windows 11",
    "browserName": "chrome",
    "browserVersion": "latest",
    "sauce:options": {
        "name": "My first test",
        "build": "Selenium 4 Build",
        "username": SAUCE_USERNAME,
        "accessKey": SAUCE_ACCESS_KEY
    }
}

# Inicializar driver
driver = webdriver.Remote(
    command_executor="https://ondemand.us-west-1.saucelabs.com/wd/hub",
    desired_capabilities=caps
)

# Executar teste
driver.get("https://www.example.com")
print(driver.title)

# Fechar driver
driver.quit()
</pre>

<h4>Configuração Cypress</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Cypress SauceLabs plugin
npm install --save-dev cypress-cloud

# cypress.config.js
const { defineConfig } = require("cypress");
const { defineBdd } = require('cypress-cucumber-preprocessor');
const createBundler = require('@cypress/webpack-preprocessor');

module.exports = defineConfig({
  e2e: {
    specPattern: "**/*.feature",
    setupNodeEvents(on, config) {
      const bundler = createBundler({});
      on("file:preprocessor", bundler);
      return defineBdd(config);
    },
    env: {
      sauceLabs: {
        region: 'us-west',
        username: process.env.SAUCE_USERNAME,
        accessKey: process.env.SAUCE_ACCESS_KEY,
        build: 'Cypress Build',
        tags: ['cypress', 'demo'],
      },
    },
  },
});

# cypress/support/e2e.js
import { defineConfig } from '@saucelabs/cypress-plugin';

defineConfig({
  sauce: {
    region: 'us-west',
    username: process.env.SAUCE_USERNAME,
    accessKey: process.env.SAUCE_ACCESS_KEY,
  },
});
</pre>

<h3>Testes Mobile no SauceLabs</h3>

<h4>Configuração Appium para Android</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Install Appium
pip install Appium-Python-Client

# Exemplo em Python
from appium import webdriver

# Configuração para Android real device
caps = {
    "platformName": "android",
    "deviceName": "Google Pixel 7",
    "platformVersion": "13.0",
    "app": "storage:filename=app.apk",  # ou storage:filename para upload
    "sauce:options": {
        "name": "Android App Test",
        "build": "Android Build",
        "username": SAUCE_USERNAME,
        "accessKey": SAUCE_ACCESS_KEY,
        "appiumVersion": "2.0.0"
    }
}

# Inicializar driver
driver = webdriver.Remote(
    "https://ondemand.us-west-1.saucelabs.com/wd/hub",
    caps
)

# Executar teste
element = driver.find_element("id", "com.example.app:id/button")
element.click()

# Fechar driver
driver.quit()
</pre>

<h3>Funcionalidades Avançadas</h3>

<h4>Parallel Testing</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Rodar testes em paralelo em múltiplos browsers
browsers = [
    {"browserName": "chrome", "platformName": "Windows 11"},
    {"browserName": "firefox", "platformName": "Windows 11"},
    {"browserName": "safari", "platformName": "macOS 13"},
    {"browserName": "edge", "platformName": "Windows 10"}
]

# Usar threading ou multiprocessing para paralelismo
from concurrent.futures import ThreadPoolExecutor

def run_test(browser_config):
    caps = {**browser_config, **common_caps}
    driver = webdriver.Remote(hub_url, caps)
    # executar teste
    driver.quit()

with ThreadPoolExecutor(max_workers=4) as executor:
    executor.map(run_test, browsers)
</pre>

<h4>Sauce Connect (Local Testing)</h4>
<pre style="background:#f5f5f5; padding:1rem; border-radius:0.5rem; overflow-x:auto">
# Download do Sauce Connect
# https://docs.saucelabs.com/dev/cli/sauce-connect/

# Iniciar Sauce Connect
./sc -u $SAUCE_USERNAME -k $SAUCE_ACCESS_KEY

# Adicionar capability
caps["sauce:options"]["tunnelIdentifier"] = "my-tunnel"

# Testar URLs internas
driver.get("http://localhost:3000")

# Sauce Connect com opções avançadas
./sc -u $SAUCE_USERNAME -k $SAUCE_ACCESS_KEY \
  --tunnel-identifier "my-tunnel" \
  --pac "proxy.pac" \
  --direct-domains "*.internal.com"
</pre>

<h3>Troubleshooting Comum</h3>

<div style="background:#fef3c7; padding:1.5rem; border-radius:0.5rem; margin:1.5rem 0; border-left:4px solid #f59e0b">
<p><strong>Problema 1: Session falha com "Invalid credentials"</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se username e access key estão corretos</li>
  <li>Regenere a access key no dashboard se necessário</li>
  <li>Verifique se não há espaços extras nas credenciais</li>
  <li>Use variáveis de ambiente para evitar hardcoding</li>
</ul>

<p><strong>Problema 2: Testes muito lentos</strong></p>
<ul style="margin:1rem 0">
  <li>Use parallel testing para reduzir tempo total</li>
  <li>Reduza resolução de screenshots/videos se não crítico</li>
  <li>Use a region mais próxima geograficamente (us-west, eu-central, ap-southeast)</li>
  <li>Desabilite capturePerformance para testes simples</li>
</ul>

<p><strong>Problema 3: App não instala no device</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se o app é compatível com o OS version</li>
  <li>Certifique-se de que o app está assinado corretamente</li>
  <li>Use storage:filename em vez de path local</li>
  <li>Verifique se o device está disponível no plano</li>
  <li>Confirme o formato correto (.apk para Android, .ipa para iOS)</li>
</ul>

<p><strong>Problema 4: Sauce Connect não conecta</strong></p>
<ul style="margin:1rem 0">
  <li>Verifique se o Sauce Connect está rodando</li>
  <li>Certifique-se de que o tunnelIdentifier está correto</li>
  <li>Verifique firewalls e configurações de rede</li>
  <li>Teste connectivity: ping saucelabs.com</li>
  <li>Use --verbose para debug do Sauce Connect</li>
</ul>
</div>

<h3>Melhores Práticas</h3>
<ul style="margin:1rem 0">
  <li><strong>Use parallel testing</strong> - reduza tempo de execução drasticamente</li>
  <li><strong>Organize com builds e tags</strong> - track testes por feature/sprint</li>
  <li><strong>Use regions apropriadas</strong> - reduza latência com region mais próxima</li>
  <li><strong>Habilite debug apenas quando necessário</strong> - aumenta custo e overhead</li>
  <li><strong>Combine com Sauce Connect</strong> - para ambientes de desenvolvimento</li>
  <li><strong>Monitore quotas e usage</strong> - evite exceder limites do plano</li>
</ul>
